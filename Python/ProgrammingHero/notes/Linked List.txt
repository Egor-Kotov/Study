Структура данных Дерево. Дерево состоит из Узлов и связей между ними. Элемент с которого начинается дерево, 
называется Корневой элемент(root). Если из Узла выходят другие Узлы их называют Потомками/Детьми. В то же 
время верхний Узел(из которого вышли Потомки) называется Родительский Узел. Если у Узла нет Потомков, 
такой Узел называется Листом.

Примеры Древесной структуры данных - генетическое древо, школа - (детектор --> замы --> учителя --> ученики).
Каждый Узел содержит следующие элементы:
1. Дату
2. Потомки

Существуют разные типы Деревьев. 1) Бинарное Древо - каждый Узел такого Древа, содержит максимум два Узла - Левый и Правый. 
2)Бинарное Поисковое Дерево - это Бинарное Дерево в котором value Левого узла меньше, чем value Родительского узла. 
А value Правого узла, больше Родительского. 4) Heap (куча): есть Max Heap - где value каждого Родительского Узла, 
больше-равно value Потомка. И Min Heap, наоборот.

Создание Древа в питоне происходит с помощью классов и функций. В начале создаётся класс Узел И Первый Корень - корневой Узел.
При добавлении последующих Узлов мы используем функцию. Входные параметры функции это сама Дата и Родитель. При добавлении 
находится Родительский Узел, создаётся Узел с Датой, и новый Узел указывается как Потомок.
Поисков Узлов. При поиске Узлов используется два типа поисков Поиск вглубь (BFS) и Поиск в глубину (DFS). 
BFS - проверяет каждый узел на одном уровне и спускается на уровень ниже и ТД. 
DFS - идёт от корневого элемента в Левый узел, потом все его потомки, потом все их потомки и ТД. 
После возвращается в Правый Узел и делает идентичную операцию.
